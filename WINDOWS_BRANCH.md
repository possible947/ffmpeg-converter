# Windows ветка проекта

Этот файл описывает назначение ветки/тега `windows` в репозитории и процесс автоматической сборки и публикации релиза для Windows (MSYS2 / MinGW‑w64).

1) Цель ветки/тега

- Тег `windows` служит триггером для CI: при push тега `windows` GitHub Actions собирает `windows_cli` в MSYS2 и автоматически создаёт GitHub Release, прикрепляя готовый `.exe` как артефакт.
- Это удобно для выпуска готовых бинарников для пользователей Windows без необходимости вручную паковать артефакты.

2) Как инициировать релиз (локально)

```bash
# Создать аннотированный тег (по желанию указывать версию вместо 'windows')
git tag -a windows -m "Windows build"
git push origin windows
```

Пуш тега `windows` запустит workflow `.github/workflows/windows-msys2.yml`, соберёт бинарник и создаст релиз на GitHub.

3) Что делает CI

- Устанавливает MSYS2 MINGW64 и пакеты (`toolchain`, `cmake`, `ffmpeg`, `jansson`).
- Конфигурирует CMake с `-G "MSYS Makefiles"` и `-DCMAKE_PREFIX_PATH=/mingw64`.
- Собирает цель `windows_cli`.
- Загружает собранный `ffmpeg_converter.exe` как артефакт run (Actions UI).
- При наличии `refs/tags/*` создаёт GitHub Release и прикрепляет exe к релизу.

4) Где смотреть результат

- Перейдите в репозиторий → Actions → выберите последний run workflow.
- В логах шагов будут подробности установки пакетов, вывода CMake и сборки.
- В правой панели run раздел `Artifacts` содержит загруженный архив; релиз с артефактами появится в Releases.

5) Локальная сборка в MSYS2 (если нужно отладить перед пушем)

В MINGW64 shell:

```bash
pacman -Syu
pacman -S --needed base-devel mingw-w64-x86_64-toolchain mingw-w64-x86_64-cmake mingw-w64-x86_64-ffmpeg mingw-w64-x86_64-jansson
mkdir build && cd build
cmake -G "MSYS Makefiles" -DCMAKE_PREFIX_PATH=/mingw64 ..
cmake --build . --target windows_cli
```

6) Подпись бинарников (рекомендация)

Если вы хотите официально распространять exe, рассмотрите цифровую подпись (SignTool) на этапе релиза; это не включено в текущий workflow.

7) Примечания безопасности

- `GITHUB_TOKEN`, используемый в Actions, имеет права на создание релиза в текущем репозитории. Не требуется дополнительный токен для типичного сценария.
- При добавлении шагов загрузки артефактов в публичные релизы убедитесь, что собранные бинарники не содержат чувствительных данных.

8) Возможные улучшения

- Добавить версионирование тегов (например `v1.2.0`) и автоматически генерировать changelog.
- Добавить шаг подписи exe и публикацию подписанного бинарника.
- Добавить тесты/скрипты smoke test под Wine в workflow перед публикацией релиза.

---

Проверка статуса Windows-версии

При проверке исходников и CMake для Windows (цель — сборка в MSYS2/MINGW64) не потребовалось правок в исходном коде или CMake: проект совместим с MSYS2 и собирается с пакетами `mingw-w64-x86_64-toolchain`, `mingw-w64-x86_64-cmake`, `mingw-w64-x86_64-ffmpeg` и `mingw-w64-x86_64-jansson`.

Если потребуется поддержка нативного MSVC — потребуется дополнительная портировка POSIX-зависимостей.
